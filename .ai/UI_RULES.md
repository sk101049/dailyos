# UI Rules

DailyOS should feel calm, focused, and useful for daily work.

## Simple, Clean UI

- Keep screens uncluttered and easy to scan.
- Prefer clear labels, predictable layout, and restrained visual styling.
- Use spacing and hierarchy instead of decorative complexity.

## Daily-Use First

- Optimize for repeated daily use, not a one-time marketing impression.
- Make the next action visible without extra explanation.
- Reduce clicks for common daily workflows.

## Dashboard-First Design

- The Dashboard is the primary entry point.
- Design the Dashboard around priorities, follow-ups, and next actions.
- Let deeper modules support the Dashboard rather than compete with it.

## Responsive Desktop-First Layout

- Design for desktop productivity first.
- Ensure layouts remain readable on tablet and mobile.
- Avoid fragile layouts that depend on exact viewport sizes.

## Tailwind And shadcn/ui

- Use Tailwind CSS for styling.
- Use shadcn/ui conventions for primitives and interaction patterns.
- Keep color, radius, and spacing aligned with the shared theme tokens.
