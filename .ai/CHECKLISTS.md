# Checklists

Use these checklists before, during, and after DailyOS changes.

## Before Coding

- [ ] Read the GitHub issue or task request.
- [ ] Confirm the requested scope.
- [ ] Inspect related files before editing.
- [ ] Check for existing patterns and conventions.
- [ ] Identify files that must not be changed.

## Before Commit

- [ ] Keep changes scoped to the issue.
- [ ] Confirm no secrets or generated artifacts are staged.
- [ ] Run relevant checks.
- [ ] Review the staged diff.
- [ ] Use the requested commit message if the issue provides one.

## Review Checklist

- [ ] Does the change solve the stated issue?
- [ ] Is the product behavior clear and simple?
- [ ] Does the UI follow DailyOS rules?
- [ ] Are TypeScript and Next.js conventions followed?
- [ ] Are docs or changelog updates needed?
- [ ] Is anything unrelated included by accident?
