# Product Principles

These principles should shape product decisions for DailyOS.

## Personal Use, Not Enterprise SaaS

DailyOS is a personal operating workspace. Prefer direct usefulness over complex admin, billing, team, or enterprise features.

## Insurance Workflow First

Prioritize insurance work: client follow-up, policy review, content planning, and daily relationship management.

## UI Before Backend

Build and validate the user workflow in the interface before adding persistence, automation, or integrations.

## Manual Before Automation

Start with manual controls and visible user choices. Automate only after the workflow is clear and repeatable.

## Human Approval Before AI Automation

AI should suggest, draft, summarize, and organize. The user should approve important actions before anything is sent, saved, or automated.
